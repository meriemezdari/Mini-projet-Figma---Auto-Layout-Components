# Mini-projet Figma – Auto Layout & Components

## Objectif
L’objectif de ce mini-projet est d’apprendre à concevoir une interface numérique structurée, cohérente utilisant correctement l’Auto Layout et le Composant.  
Vous travaillerez sur une interface simple mais réaliste, proche d’un cas professionnel.

---

## Contexte du projet
Vous êtes designer UI junior pour un projet numérique.  
On vous demande de concevoir une interface simple, fonctionnelle, claire.

---

## Type d’interface à concevoir
**Page d’accueil :**
- Présentation générale du service  
- Mise en avant d’un contenu principal  
- Un ou deux appels à l’action (CTA)  
- Une seule page est attendue (pas de site complet)  

---

## Domaine / thématique (au choix)
Choisissez un seul domaine, voici quelques exemples :
- Site de cuisine / recettes  
- Musée ou lieu culturel  
- Jeu vidéo / plateforme gaming  
- Bibliothèque ou médiathèque  
- Formation en ligne / école  
- Service local éco-responsable  
- Site de finance  

Le domaine doit rester réaliste et compréhensible par tous.  
N'hésitez pas à utiliser des références de sites sur **Dribbble** ou **Behance** pour vous inspirer.

---

## Contraintes techniques (obligatoires)

### Auto Layout
- Tous les boutons doivent utiliser l’Auto Layout  
- Les cartes, listes ou formulaires, etc., doivent être organisés avec Auto Layout  
- Les espacements doivent être gérés automatiquement (padding / gap)  

### Composants
Vous devez créer au minimum 2 composants :  
1. Bouton (texte variable)  
2. Carte ou bloc de contenu  
3. Header, footer ou élément de navigation  

Chaque composant doit être réutilisé au moins deux fois.

---

## Consignes de conception
- Interface simple et lisible  
- Hiérarchie visuelle claire  
- Peu de scroll  
- Texte lisible (≥16px)  

---

## Livrables attendus
- Nomenclature **Nom_Prenom** sur le dossier et chaque fichier obligatoire  
- Une page Figma avec :  
  - Les composants visibles sur la page et calques  
  - Les auto-layout visibles également sur la page et calques  
  - L’interface finale assemblée  
- Une note d’intention (5 à 10 lignes) :  
  - Cette note doit expliquer votre démarche et contenir obligatoirement :  
    - Le domaine / thématique du projet  
    - Vos sources d’inspiration (sites, applications, références visuelles)  
    - Vos choix de navigation et de structure (organisation des éléments, CTA, parcours)

---

## Note d’intention
**Domaine / thématique du projet :**  
Ce projet a pour thématique le **tourisme et la découverte culturelle**. Il s’agit d’une page d’accueil d’un site web présentant la **Tunisie comme destination de voyage** à travers ses sites historiques, ses paysages et son patrimoine.

**Sources d’inspiration :**  
Je me suis inspirée des sites **voyageursdumonde.fr** et **lonelyplanet.fr** pour l’organisation visuelle claire ainsi que l’usage de grandes images immersives. J’ai choisi des couleurs claires qui représentent bien la Tunisie.

**Choix de navigation et structure :**  
La navigation est organisée autour de **quatre rubriques principales** : Accueil, Destinations, Culture, Contact pour une accessibilité rapide.  
Le parcours utilisateur commence par une **bannière d’accroche visuelle** avec un appel à l’action (« EXPLORER »), puis propose une sélection de destinations connues (Sidi Bou Saïd, Tozeur, Djerba) sous forme de cartes cliquables.  
Les **auto-layout sur Figma** ont été utilisés pour assurer la cohérence des espaces, des alignements et une adaptation fluide aux différents écrans. Les **CTA** sont contrastés pour guider l’utilisateur vers plus de contenu.
